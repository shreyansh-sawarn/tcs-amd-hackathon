TCS & AMD AI Hackathon
Innovate. Build. Accelerate with AI 

The TCS & AMD AI Hackathon is an innovation‑driven challenge designed to empower TCS developers and engineers to build real‑world, enterprise‑ready AI solutions using modern AI frameworks and AMD‑powered accelerated computing. Focused on applied AI—including GenAI, RAG, AI agents, vision, speech, and multimodal systems—the hackathon brings together AMD’s AI platforms, tools, and learning resources with TCS’s engineering and innovation ecosystem, enabling participants to move rapidly from ideas to working prototypes, collaborate with peers and mentors, and showcase impactful solutions to business and technology leaders.

Hackathon Challenges:
Come build the next generation of AI agents and high-performance applications - powered by AMD.

Refer the tracks below and choose your usecase:

Track 1 – Agents

Description:  Single or multiple agents orchestrating different tasks and communicating with each other. Can include RAG pipelines, agents, regression models, and chatbots to drive efficiency in BU-specific tasks. Use public datasets to build use cases using open-source stacks (Llama, Deepseek, Qwen, Graph AI, predictive ML models). Focus on productivity, efficiency (tokens used, latency), and accuracy.


Track 2 – Multimodal

Description: Applicable for complex environments like manufacturing, networking, industrial, and pharma. Involves multiple models working together to analyze, react, and enable decision-making. Build combined pipelines (vision, reasoning, speech, video analysis models) to create effective end-to-end solutions


Track 3 – Fine Tuning (Advanced Users) 

Description: Open-source LLMs may hallucinate in domain-specific use cases such as code generation or telecom/pharma expertise.  Apply fine-tuning techniques like PEFT, GRPO, etc., to adapt models for specific domains/tasks. Demonstrate measurable improvements in accuracy and performance after fine-tuning

What to Submit?

Each team must submit the code files, a 3–5 slide presentation (PDF or PPT) and demo recording as part of the final submission. The PPT/PDF should follow the following structure - the template can be downloaded Here

Slide 1 – Basic Information 

    Team name 

    Names of the team members / individual / role in the team for the Hackathon (Research, coding, building the business case, presentation, etc) 

    Usecase detail

    Short Description 

Slide 2 – Problem & Context 

    Problem statement 

    Target users / stakeholders 

    Why this problem matters (business or operational relevance) 

    Mapped hackathon challenge 

Slide 3 – Solution Overview 

    Solution architecture / workflow diagram 

    AI approach used (e.g., GenAI, RAG, agents, vision, speech, multimodal) 

    Key technologies / frameworks leveraged  

    What was built during the hackathon 

Slide 4 - Model Insights

    Models used 

    Dataset(s) used (if fine-tuning) 

    Training time (if fine tuning) 

    Number of tokens used for a couple of example scenarios 

    End-to-end latency 

    GPU usage including memory (gives a sense of right sizing) 

Slide 5 – Impact & Demo Summary 

    Expected impact or value (efficiency, productivity, scale, experience) 

    Key differentiators / innovation 

    Demo flow overview (what the jury should notice) 

    Future extension (Planned if any)

Guidelines to be followed

    All development must be done during the hackathon period 

    Teams may use open-source libraries and frameworks, but must clearly state what was built during the hackathon 

    Code reuse from prior work must be disclosed 

    Submissions must include a recorded demo, presentation, source code (as .doc/.pdf) used 

    Any form of plagiarism or rule violation may result in disqualification 

    Judges’ decisions will be final
